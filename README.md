# معمل المنظفات — حاسبة تكاليف المواد وأسعار المنتجات

موقع بسيط بـ **Next.js** لإدارة أسعار المواد الأولية في معمل منظفات، وحساب **سعر الكيلو**
لكل منتج تلقائياً عند تغيّر أسعار المواد.

## الفكرة
1. عرّف موادك الأولية مرّة واحدة (ملح، ماء، زفت...) وسعّر كل وحدة (كيلو / لتر).
2. أنشئ منتجاً (خلطة) مثل «دواء غسيل»، وحدّد المواد وكمياتها والكمية المنتجة.
3. يحسب الموقع التكلفة الإجمالية وسعر الكيلو تلقائياً.
4. عند تغيّر سعر أي مادة، عدّله من مكان واحد فيتحدّث رأس مال كل المنتجات فوراً.

> مثال: ملح 10$/كغ، ماء 1$/ل، زفت 20$/كغ.
> خلطة = 3 ملح + 5 ماء + 1 زفت = 30 + 5 + 20 = **55$**، تُنتج 10 كغ ⇒ **5.50$/كغ**.

## التقنيات
- Next.js (App Router) + TypeScript + Tailwind CSS — واجهة عربية حديثة متجاوبة مع الموبايل (RTL).
- جلسات دخول موقّعة (JWT في كوكي httpOnly).
- تخزين بسيط بدون قاعدة بيانات:
	- محلياً: ملف JSON في `data/store.json`
	- على Vercel: ملف JSON داخل `Vercel Blob`

## التشغيل


للبناء والإنتاج:

## الحساب الثابت
يُقرأ مباشرة من ملف `.env` (`SEED_EMAIL` / `SEED_PASSWORD`). الافتراضي:

- البريد: `test@test.com`
- كلمة السر: `test1234`

غيّرها في `.env` فقط، وغيّر `AUTH_SECRET` لقيمة عشوائية طويلة.

## Vercel
- لا ترفع ملف `.env` إلى GitHub.
- أضف القيم نفسها داخل إعدادات Vercel في `Environment Variables`.
- للحفظ الدائم بدون قاعدة بيانات، يجب ربط `Vercel Blob` بالمشروع نفسه. بدون هذه الخطوة ستظهر رسالة أن التخزين غير مهيأ ولن تُحفظ المواد أو المنتجات.
- الخطوات الصحيحة:
  1. افتح مشروعك في Vercel.
  2. ادخل إلى `Storage`.
  3. اختر `Create Database` ثم `Blob`.
  4. بعد إنشاء الـ Blob اختر `Connect Project` أو `Attach to Project` لنفس المشروع.
  5. تأكد أن Vercel أضاف أحد المتغيرين للمشروع: `BLOB_READ_WRITE_TOKEN` أو `BLOB_STORE_ID`.
  6. أعد النشر `Redeploy` بعد الربط.
- عند الربط، يضيف Vercel متغيرات Blob اللازمة تلقائياً للمشروع في أغلب الحالات.
- المتغيرات التي تضبطها أنت يدوياً:
  - `AUTH_SECRET`
  - `SEED_EMAIL`
  - `SEED_PASSWORD`
- بعد ربط Blob، سيحفظ التطبيق المواد والمنتجات تلقائياً على Vercel.
- إذا أردت الاعتماد على OIDC بدلاً من التوكن المباشر، يكفي وجود `BLOB_STORE_ID` بعد ربط المشروع على Vercel.

## ملاحظات
- العملة بالدولار `$`، والكميات تدعم الكسور (مثل `1.25`).
- لا تحفظ الأسرار داخل GitHub؛ استخدم `.env` محلياً و`Environment Variables` داخل Vercel.
- إذا لم تربط Blob على Vercel، فلن يكون التخزين دائماً هناك ولن تعمل عمليات الإضافة والتعديل في البيئة المنشورة.
